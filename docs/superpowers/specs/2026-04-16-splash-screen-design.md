# 开屏动画设计：医生的智能体空间

## 背景与目标

在 Web 应用首次渲染后，展示一段 1.5 秒的开屏动画，用于传达产品定位与情绪基调，并在结束后进入现有路由内容。

目标：

- 每次打开/刷新页面均展示开屏动画
- 动画总时长约 1.5 秒，节奏利落
- 文案层级为“大标题 + 小字”，背景为柔和渐变
- 技术实现尽量复用现有技术栈（React + Tailwind + framer-motion）

非目标：

- 不做原生层/OS 启动屏
- 不做仅首次访问或版本控制展示
- 不引入新动画库（如 Lottie/Rive）

## 方案选择

采用方案 A：React 层开屏覆盖（Overlay）+ framer-motion 动效。

原因：

- 改动集中且与现有工程结构一致
- 动画时序与退场可控
- 后续可扩展为“仅首次访问/版本展示”或引导页

## 交互与视觉规范

### 文案与层级

主标题（Heading）：

- 医生的智能体空间

副标题（Subheading）：

- 让健康触手可及，做自我健康第一责任人

辅助文案（Supporting）：

- 生了病，不担心
- 少生病，更健康

### 排版布局

- 画面居中对齐，文本居中
- 主标题字号明显高于副标题（约 1.6–2.2 倍）
- 副标题行宽限制（避免过长导致阅读困难），两侧留白充足
- 底部两行与上部文字保持明显间距，权重更轻（更淡/更小/字重更低）

### 背景

- 米白到浅蓝的柔和渐变
- 可叠加极轻微雾感（低对比度径向高光）以提升质感

### 动效节奏（总时长约 1.5s）

- 入场（0–0.9s）：整体淡入；主标题轻微上移并从略小尺寸到 1；其余文案错峰出现（stagger）
- 停留（0.9–1.2s）：静止阅读窗口
- 退场（1.2–1.5s）：整体淡出（可伴随轻微缩小）

### 无障碍与动效降级

- 遵从 `prefers-reduced-motion`：若用户系统偏好减少动效，改为更短的淡入淡出并减少位移/缩放

## 技术设计

### 组件结构

- 新增 `SplashScreen` 组件，渲染覆盖层与动效容器
- 在应用根组件中控制展示/隐藏（定时 1.5s 关闭）

### 状态与时序

- `showSplash: boolean` 初始为 `true`
- `useEffect` 启动后设置定时器 `1500ms` 关闭
- 使用 `AnimatePresence` 负责退场动画，退场完成后移除 DOM

### 路由关系

- Splash 为路由之上的覆盖层，不改变现有 `react-router-dom` 配置
- Splash 展示期间仍允许 Router 渲染（被遮罩覆盖），避免延迟业务初始化

### 样式约束

- 优先使用 Tailwind class
- 颜色尽量使用现有全局变量语义（医疗蓝等）或与现有背景色一致

## 验收标准

- 每次打开/刷新页面进入站点，均先出现开屏动画
- 开屏文案与层级符合“大标题 + 小字”，背景为柔和渐变
- 动画总时长约 1.5 秒，结束后可正常操作既有页面与导航
- `prefers-reduced-motion` 下动画显著简化但仍可读

## 变更范围

- 修改：`src/App.tsx`
- 新增：`src/components/SplashScreen.tsx`（或同等位置）
- 可能修改：全局样式/字体类名仅用于复用现有主题（视实现而定）
